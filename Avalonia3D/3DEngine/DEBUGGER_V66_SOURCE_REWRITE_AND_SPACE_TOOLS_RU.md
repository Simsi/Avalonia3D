# 3DEngine Debugger v66 — source rewrite и инструменты пространства

v66 переводит экспериментальную генерацию из companion-файла в реальную перезапись исходника.

## Экспорт кода в исходник

Кнопка `Export code to source (experimental)` теперь пытается найти исходный `.cs` файл класса, открытого в debugger-е.

Источник определяется по:

- `--project` / `--host-project`, если previewer запущен из VSIX;
- `--assembly`, если project path недоступен;
- имени класса из `--type`.

Debugger запоминает найденный файл, имя класса, строку класса и наличие метода:

```csharp
protected override void Build(CompositeBuilder3D builder)
```

Если `Build(...)` найден, заменяется только этот метод. Если метода нет, выполняется полная замена блока класса минимальной заготовкой `CompositeObject3D`.

Перед записью показывается предупреждение:

> Эта операция полностью заменит код изменяемого класса. Рекомендуется использовать для быстрого прототипирования 3D представления объекта перед последующим наполнением класса.

Перед перезаписью создаётся backup:

```text
<file>.3ddebugger.bak
```

## Что экспортируется

Приоритет экспорта:

1. объекты, созданные в debugger/workbench;
2. direct children текущего previewed `CompositeObject3D`;
3. выбранный объект;
4. root-объекты сцены.

В экспорт не попадают служебные guide-объекты пространства.

## Инструменты пространства

Добавлены:

- RGB basis axes: X красный, Y зелёный, Z синий;
- ground grid на плоскости `Y = 0`;
- readout камеры;
- координаты pointer-а в viewport;
- пересечение pointer ray с плоскостью `Y = 0`.

Это нужно для ручной расстановки объектов и понимания ориентации сцены.

## Ограничения

- Это экспериментальный rewrite, не полноценный Roslyn/VS code action.
- Сложные nested/partial/generic классы могут требовать ручной проверки diff-а.
- Для production-использования следующий шаг — VSIX-side patch preview через Roslyn syntax tree и подтверждение diff-а в Visual Studio.
