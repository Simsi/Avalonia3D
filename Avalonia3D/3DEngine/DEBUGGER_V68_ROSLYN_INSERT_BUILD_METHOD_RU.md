# Debugger v68 — Roslyn вставка Build-метода

Изменение v68 исправляет поведение экспериментального экспорта исходников.

## Старое поведение

Если целевой класс был найден, но метод `Build(CompositeBuilder3D builder)` отсутствовал, exporter заменял весь класс fallback-заготовкой. Это было слишком разрушительно: вместе с классом могли потеряться поля, конструкторы, методы, атрибуты и пользовательская логика.

## Новое поведение

Если класс найден:

- при наличии `Build(...)` заменяется только этот метод;
- при отсутствии `Build(...)` в класс добавляется только новый сгенерированный метод `Build(...)`;
- остальной код класса сохраняется.

Если класс не найден, экспорт завершается ошибкой. Автоматическая замена всего класса больше не используется для случая «метод не найден».

Перед записью по-прежнему создаётся backup-файл:

```text
<source>.3ddebugger.bak
```
